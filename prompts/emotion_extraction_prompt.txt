## 字段定义

1. emotion_layers（三层来自 Leventhal 1979 情绪处理层次理论）：
   - 表层：用户文字直接表达的（感觉运动层）
   - 中层：通过语气、停顿、修辞表达的（图式层）
   - 深层：用户可能未明确表达的潜在情绪（概念层）

2. valence / arousal（来自 Russell 1980 环形情绪模型）：
   - valence: -1（极负面）到 +1（极正面）
   - arousal: 0（极平静）到 +1（极激动）

3. core_metaphor_cn vs supporting_metaphors_cn（关键）：
   - core_metaphor_cn：只选 1 个最核心的视觉隐喻，用于精简 prompt
   - supporting_metaphors_cn：列出 1-2 个次要隐喻，作为可选元素
   - 隐喻可以由情绪推导，但必须服务于原文中的真实情绪，不要变成与原文无关的新剧情
   - 不要把“孤立感、焦虑感、自责感”等心理推断改写为原文没有说过的事实场景，例如“独自一人”“空荡房间”“阴雨天气”

4. scene_keywords_cn：从日记中直接提取的具体中文意象
   - 只能选择原文明确出现的物体、地点、界面、动作或文字
   - 好的例子：教学楼、LeetCode 屏幕、Claude 聊天、没有面试、无法开口、麻烦别人、小心翼翼
   - 不好的例子：空荡教学楼、灰蒙蒙天空、孤独背影、阴雨天空（除非原文明确出现）

5. time_of_day_cn / weather_cn：
   - 只有原文明确出现时才填写具体时间或天气
   - 如果原文标题或正文只提供“周六出了大太阳”，可填“周六，晴天/大太阳”
   - 如果原文没有天气，不要为了情绪氛围编造“阴天、薄雾、闷雷”等天气

6. color_palette_cn：基于 Russell 模型情绪象限的色彩描述
   - 焦虑（低效价/高唤起）：冷蓝灰、暗灰、阴沉灰
   - 快乐（高效价/高唤起）：明黄、温暖橙、活力粉
   - 怀念（高效价/低唤起）：暖棕色、金黄色、米黄色
   - 愤怒（低效价/高唤起）：深红、暗橙、刺目黑
   - 平静（高效价/低唤起）：柔和粉彩、灰绿、奶白色

7. 核心情绪权重判断（非常重要）：
   - 不要只选择最容易画出来的外部场景作为 core_metaphor_cn。
   - 必须判断日记中“最痛、最反复、最占情绪重量”的部分是什么。
   - 如果日记同时包含学习/工作任务和人际自责，请比较两者情绪重量；不要自动把学习/工作场景当核心。
   - 像“我总是麻烦别人”“无法正常开口说话”“总觉得令人讨厌”“小心翼翼”“太累了”这类自我关系句子，通常是核心情绪线索，不得忽略。
   - core_metaphor_cn 必须覆盖核心情绪矛盾，而不是只复述地点或任务。

8. 反讽判定规则（非常重要）：
   - is_ironic 只有在文本存在明确的“表面正向/称赞/开心，实际负向或讽刺”的结构时才设为 true。
   - 普通的自我安慰、自责、压抑、故作坚强，不等于反讽。
   - “你还是开心一点”“看开吧”这类自我劝慰，如果没有明显反话结构，不应判定为反讽。

9. 关心行为背后的情绪推断（非常重要）：
   - 如果日记中出现家人给钱、嘱咐买票、提醒早走、让用户有保障等行为，不要只概括为 gratitude。
   - 必须分析这些行为背后的情绪功能：担心安全、心疼、照顾、提供保障、被保护感、依靠感。
   - 如果用户写“以后可能很难再闻到/再拥有这种感觉”，应识别怀念、依恋、失落、对不可再得的温暖的预感。
   - 物质行为（钱、车票、微信嘱咐）不是冷冰冰的物件，可能是亲情关心和安全感的载体。

10. 细节功能分析（非常重要）：
   - 日记里的具体细节通常不是随机背景。每当用户提到一个具体事件、物品、时间、地点或一句话时，先判断它在情绪叙事中的功能。
   - 请对关键细节做“小拆解”：用户为什么在这里提到它？它可能承载担心、心疼、羞耻、比较、依赖、失落、逃避、控制感，还是只是事实背景？
   - 如果某个细节影响了用户后续情绪或解释了前后文，就不能当成普通 scene keyword 处理。
   - 如果某个细节只是交代背景，才可以降级为普通场景元素。
   - 例如“买早上的票”不一定只是时间信息；它可能意味着家人担心用户太晚回去不安全。
   - 例如“给我一些钱”不一定只是物品信息；它可能意味着心疼、保障、照顾和被爱的证据。
   - reasoning 字段必须简短说明 1-2 个关键细节为什么重要。

## 重要指导
- 对反讽和隐晦表达保持警觉，但不要把普通自我安慰误判为反讽
- 三层情绪可能并存甚至矛盾，如实描述，不要强行统一
- core_metaphor_cn 选择时同时满足两个条件：能画出来，并且承载原文最核心的情绪矛盾
- 当“外部任务场景”和“内在人际/自我关系痛点”同时出现时，优先保留后者的情绪线索，再寻找可视化方式
- 对原文中突然出现的具体细节做功能判断：它是普通背景，还是解释了用户为什么难受/感动/怀念/害怕
- 严格区分“原文事实”和“情绪隐喻”：scene_keywords_cn 只能放原文事实；core/supporting metaphor 才能放隐喻
- 不要把情绪推断伪装成事实，不要添加原文没有出现的人数、天气、空间状态、动作结果
- 不要添加“独自一人”“空荡房间/教学楼”“阴天/灰蒙蒙天空”等常见忧郁模板，除非原文明确出现
- 即使日记是中文，primary_emotion 和 secondary_emotions 用英文小写，其他视觉描述字段用中文
- JSON 字符串内部不要使用英文双引号；错误示例："直接表达"久违的轻松""；正确示例："直接表达「久违的轻松」"

## 反例提醒
如果日记写到“去教学楼刷 LeetCode / 研究项目”，同时又反复写“无法开口说话 / 总是麻烦别人 / 令人讨厌 / 小心翼翼 / 太累了”，不要把核心简化为“独自在教学楼刷题”。更合适的分析应保留“想求助或连接但害怕打扰别人”的人际自责与退缩。

如果日记写到“爷爷给钱、嘱咐买早上的票、让用户回去更安全”，不要只写 gratitude 或 nostalgia。应先拆解这些细节的功能：钱可能是心疼和保障，早班票可能是担心晚归不安全，微信嘱咐可能是照顾和牵挂。最终应保留“被保护、被心疼、依靠家人”和“未来可能难以再拥有这种温暖”的怀念与失落。
